# Low-Level Design (LLD): RAG-Based Customer Support Assistant

## 1. Module-Level Design
The system is organized into a modular `backend` package to ensure separation of concerns.

- **`backend/ingestion/`**:
    - `pdf_loader.py`: Handles file I/O and PDF extraction.
    - `chunker.py`: Implements recursive character splitting logic.
    - `embedder.py`: Initializes the HuggingFace embedding singleton.
- **`backend/vectorstore/`**:
    - `chroma_store.py`: Manages ChromaDB collection initialization and document upserts.
- **`backend/retrieval/`**:
    - `retriever.py`: Configures the vector store as a LangChain retriever object.
- **`backend/llm/`**:
    - `groq_llm.py`: Manages Groq client initialization and prompt templating.
- **`backend/workflow/`**:
    - `graph.py`: Defines the LangGraph `StateGraph` and compiles it.
    - `nodes.py`: Contains the logic for the `retrieve` and `generate` nodes.
    - `state.py`: Defines the TypedDict for the shared graph state.
    - `router.py`: Implements the conditional routing logic.
- **`backend/hitl/`**:
    - `escalation.py`: Handles the logic for human-in-the-loop handovers.

## 2. Data Structures
### State Object (`GraphState`)
The system uses a stateful approach where a single object is passed and modified by each node.
```python
from typing import TypedDict, List

class GraphState(TypedDict):
    question: str      # The original user query
    documents: List    # List of retrieved LangChain documents
    answer: str        # The LLM generated answer
    confidence: float  # Score (0.0 to 1.0) indicating system confidence
```

### 1. Document & Chunk Representation
- **Document Object**: Standard LangChain `Document` class.
    - `page_content`: String (The raw text of the chunk).
    - `metadata`: Dictionary (Includes `source` file path and `page` number).
- **Chunk Format**: Text segments generated by `RecursiveCharacterTextSplitter`.
    - **Size**: ~1000 characters.
    - **Overlap**: 200 characters to prevent loss of semantic context at boundaries.

### 2. Embedding Structure
- **Model**: `sentence-transformers/all-MiniLM-L6-v2`.
- **Dimensions**: 384-dimensional floating-point vector.
- **Storage**: Vectors are indexed in ChromaDB using HNSW (Hierarchical Navigable Small World) for efficient similarity search.

### 3. Query-Response Schema
- **Query**: String-based natural language input.
- **Response**: JSON object containing the final answer string and metadata.

## 3. Workflow Design (LangGraph)
### Nodes & Transitions
1. **`retrieve`**: Takes `question`, updates `documents`.
2. **`generate`**: Takes `documents` + `question`, updates `answer` and `confidence`.
3. **`human`**: Activated if `confidence < 0.5`. Updates `answer` with a support ticket message.

### Conditional Routing Logic
The `route_decision` function acts as a traffic controller after the `generate` node:
- **Criteria**: Confidence score based on document availability.
- **Logic**:
  ```python
  if state["confidence"] < 0.5:
      return "human"
  else:
      return "output"
  ```

---

## 4. HITL Design
- **Trigger**: Triggered automatically when the retriever fails to find relevant chunks or the LLM output is flagged as low-confidence.
- **Integration**: The `human_escalation` node intercepts the flow, preventing the LLM from hallucinating an answer when it lacks data. Instead, it provides a structured "Ticket Created" response.

## 5. API / Interface Design
### Endpoint: `POST /query`
- **Request Body**:
  ```json
  {
    "question": "How do I reset my password?"
  }
  ```
- **Response Body**:
  ```json
  {
    "answer": "To reset your password, go to settings..."
  }
  ```

---

## 6. Error Handling
- **Missing Data**: Handled by the `confidence` score in the routing layer.
- **No Relevant Chunks**: The `retrieve_node` returns an empty list, which sets confidence to 0.3, triggering escalation.
- **LLM Failure**: Wrapped in the Groq client's internal retry logic and FastAPI's standard exception handlers.
